**Version:** 5.0 — Canon Edition

---

# Purpose

This document defines the major production milestones for **Yggdrasil Saga**.

Each milestone represents a stable, playable version of the project and provides clear development goals for every
department.

---

# Milestone Philosophy

Every milestone must satisfy five requirements:

- Stable
- Playable
- Canon Accurate
- Performance Tested
- QA Approved

Incomplete or unstable milestones should never be merged into the primary development branch.

# Exit Criteria

A milestone cannot be marked complete until:

- All planned work is finished.
- No Critical bugs remain.
- QA signs off.
- Canon matches CANON_REFERENCE.md.
- Performance targets are met.

---

# Success Metrics

Project success is measured by:

- Stable gameplay
- Consistent canon
- Complete documentation
- Reliable performance
- High-quality player experience

---

# Canon Principles

- Documentation is completed before implementation whenever practical.
- Canon consistency takes priority over feature additions.
- Ratatoskr is present from the Prologue onward.
- The Ten Rune Covenant drives progression through every milestone.
- Every milestone must produce a playable build.
