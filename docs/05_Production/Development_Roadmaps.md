# Purpose

This document defines the long-term development roadmap for **Yggdrasil Saga**.

It provides a structured plan from pre-production through post-launch support and serves as the primary reference for
project scheduling.

---

# Development Philosophy

Development follows five guiding principles:

- Canon First
- Playable Milestones
- Feature Stability
- Continuous Testing
- Nintendo DS Hardware Accuracy

Every milestone must result in a stable, playable build.

# Success Criteria

The project is considered complete when:

- All Ten Realms are playable.
- Every class is functional.
- All three ending routes are complete.
- Ratatoskr accompanies the Hero throughout the adventure.
- The Ten Rune Covenant functions correctly.
- All documentation matches CANON_REFERENCE.md.
- The game runs reliably on Nintendo DS hardware.

---

# Long-Term Vision

Yggdrasil Saga should deliver a classic Nintendo DS JRPG experience while telling an original story inspired by Norse
mythology.

Every system should reinforce the central question:

> **Should the current age be preserved, renewed, or should the cycle itself be broken?**

---

# Canon Principles

- CANON_REFERENCE.md is the highest authority.
- Story consistency takes priority over feature expansion.
- Ratatoskr remains the Hero's permanent companion.
- The Ten Rune Covenant is the foundation of gameplay and narrative.
- Every development milestone must preserve canon consistency.
